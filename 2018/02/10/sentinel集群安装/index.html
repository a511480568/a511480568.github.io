sentinel集群安装

发表于 2018-02-10 | 更新于 2018-02-10 | 分类于 高并发缓存

之前的文章已在四台机器中部署了redis，现在其中三台部署sentinel集群

配置sentinel相关参数

  • 创建目录/etc/sentinel 用于存放sentinel配置文件

mkdir /etc/sentinel

  • 创建目录/var/sentinel/5000 用于存放数据

mkdir -p /var/sentinel/5000

从安装的redis目录中copy出一份sentinel.conf文件，修改其中的参数，然后传输到/etc/sentinel目录中

配置如下：

port 5000

bind 192.168.0.117

dir /var/sentinel/5000

sentinel monitor mymaster 192.168.0.117 6379 2

sentinel down-after-milliseconds mymaster 30000

sentinel failover-timeout mymaster 60000

sentinel parallel-syncs mymaster 1

注意：dir的配置一定要和创建的/var/sentinel/5000一致，要不然在启动sentinel时会报错：Can’t chdir to ‘/var/sentinel/5000’: No such file or directory

启动sentinel

redis-sentinel /etc/sentinal/5000.conf

查看哨兵状态

redis-cli -h 192.168.0.117 -p 5000

sentinel master mymaster

SENTINEL slaves mymaster

SENTINEL sentinels mymaster

SENTINEL get-master-addr-by-name mymaster
