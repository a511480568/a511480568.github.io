hexo博客搭建

发表于 2018-02-10 | 更新于 2018-02-10 | 分类于 hexo

缘起

最开始写博客都是在一些oschina、博客园等一些地方，慢慢的想自己鼓捣一下，我觉得作为一个程序猿需要有这种鼓捣的精神，要不然总是看最终会发现自己的眼高手低，当你自己动手去做一些事的时候会发现有些事没有想的那么简单，在做的过程中会会碰到各种各样的问题，我在解决这些问题的时候是快乐的，因为这些问题可以让我学到很多东西。

废话有点多了，有自己想搭建博客的想法在网上找了很多大神的博客，第一次选择了ghost博客，为此我专门买了一年的vps，经过查各种资料，终于在vps上搭建了属于我自己的博客，当时还是比较兴奋的，毕竟是有了自己的博客，接下来是写博客，慢慢的觉得ghost写博客不是那么方便了，由于ghost有后台系统，博客都是在后台系统写好之后发布到vps中，这个后台系统我用不习惯，没办法，只能再转战战场，经过一箩筐的搜索找到了hexo，我用这个我的想法是它更灵活，主题繁多，可玩性更大。当然每个人的想法不一致，废话不多说下面的搭建hexo博客的步骤。

安装git

程序猿应该都多多少少用过git吧，hexo就是把文章放在git上，我下载的是：Git-2.9.0-64-bit（Windows系统），具体安装根据步骤来就可以了，下载地址：git下载地址

安装node

因为hexo发布上传等是用node来进行的，我安装的node版本是6.0，具体安装也非常简单，只需要next就可以了。node下载地址

安装hexo

打开git Bash，输入命令：npm install -g hexo

接下来就是等待安装hexo

hexo初始化

找一个盘符，创建文件夹hexo（名字可以自己随便起），进入hexo文件夹，在git Bash窗口中进入hexo目录，执行命令：hexo init，初始化hexo，初始化完成之后该文件夹下就会多出很多文件和文件夹，其中_config.yml是配置博客的相关内容，比如：博客的名称、作者、博客的主题、博客的语言等等

hexo主题安装

hexo的主题非常多也非常好看，其中很典型的一个主题是next主题，这个主题漂亮而且非常适合初学者，输入命令：git clone https://github.com/iissnan/hexo-theme-next themes/next（要在hexo目录下），clone完成进入theme目录就会看到next的文件夹，里面的_config.yml是next主题相关配置，可以修改其中的参数来变换主题的样式，关于next主题的样式网上非常多，可以找找看看，自己修改下

hexo本地发布浏览

next主题下载完成之后，在hexo目录中的_config.yml文件中修改主题为next，然后执行以下两个命令：

hexo g

hexo s

执行完成以后，在浏览器输入localhost:4000就可以浏览你刚刚部署成功的hexo页面了

hexo发布到git

接下来就是将hexo发布到git上，在git上浏览你初始安装好的hexo

  • 首先需要在你的git上创建一个repository，名字一定要和git的名字一致：xxx/xxx.git.io

  • 创建好之后修改hexo目录下的_config.yml文件

  • 安装hexo-deployer-git自动部署发布工具: npm instal lhexo-deployer-git –save

  • 输入命令发布到git上：hexo clean && hexo g && hexo d

访问hexo

在浏览器中输入：https://xxx.github.io/ 就可以看到刚刚发布上去的hexo，接下来就可以写文档然后本地浏览没问题就可以发布上去了（文章需要写在source目录中的_post目录中，文章使用的是markdown的语法）

如何在多台电脑上编辑hexo文章

由于hexo是保存在本地的，而发布到git上的只是文章而已，如果想在另一台电脑上写文章并发布到git上，没有hexo初始化的那些文件是不行的，为此我们可以在git上专门建立一个一个仓库：hexo用来存放hexo原始文件，这样即使以后换了电脑只需要将hexo仓库clone下来就可以了
