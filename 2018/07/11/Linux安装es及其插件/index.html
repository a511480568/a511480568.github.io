Linux安装elasticsearch

发表于 2018-07-11 | 更新于 2018-07-11 | 分类于 其他

经过两天的折腾终于把es装好了，我第一次安装成功之后当再次启动es的时候报错：找不到elasticsearch.log，很奇怪，最后我把es装载home目录下就OK了，原因现在还没找出。。

安装elasticsearch

安装jdk

首先要安装es需要的jdk版本至少是1.8，至于在Linux中安装es不是本文重点，各位可以自行百度

下载es版本并解压

大家可以在es官网下载相应的版本然后传到服务器中，我是放在了/home/es目录下，使用tar -zxvf进行解压

  • 修改配置参数

重新启动es这时可能会发现报错：提示虚拟内存太小，最大文件描述进程太小

我们需要做以下修改：

  • max file descriptors

修改/etc/security/limits.conf，添加或者修改（此时不要忘了把用户切回root）

  • max virtual memory
    修改/etc/sysctl.conf 添加

保存后执行sudo sysctl -p /etc/sysctl.conf 使之生效

其中：

soft nproc:可打开的文件描述符的最大数（软限制）
hard nproc:可打开的文件描述符的最大数（硬限制）
soft nofile:单个用户可用的最大进程数（软限制）
hard nofile:单个用户可用的最大进程数（硬限制）

安装elasticsearch-head插件

下载

从git上下载到源码elasticsearch-head

我这边是下载到/usr/local/elastic-head目录中

安装grunt

注意：安装grunt是需要node支持的

修改配置

  • 首先我们需要修改elasticsearch-head目录中的Gruntfile.js，head是默认监听的127.0.0.1,我们需要修改成我们服务器的IP地址，使用vi Gruntfile.js
  • 修改_site目录中app.js

修改_site目录中的app.js是为了启动head插件时能够关联到服务器上的es，如果不修改默认关联的是http://127.0.0.1:9200

  • 修改elasticsearch.yml支持跨域

接着我们修改elasticsearch.yml文件使其能够支持跨域

启动

这是我们重新启动es 并启动elasticsearch-head

这是我们在浏览器中输入地址：http://176.122.xx.xxx:9100/ 会发现我们已经关联了es，可以看到es的健康状态

下载kibana插件

去官网下载kibana插件kibana

上传至服务器并解压

将下载好的插件传到服务器上，我是放在/usr/local目录中，然后使用tar命令进行解压

修改配置文件
