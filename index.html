Redis数据结构之列表相关命令

发表于 2018-08-22 | 更新于 2018-08-22 | 分类于 redis

这篇文章主要介绍的是Redis五大数据结构中的列表常用的相关的命令

一些常用的列表命令

命令 行为
RPUSH RPUSH key-name value [value…] 将一个或多个值推入列表的右端
LPUSH LPUSH key-name value [value…] 将一个或多个值推入列表的左端
RPOP RPOP key-name 移除并返回列表最右端的元素
LPOP LPOP key-name 移除并返回列表最左端的元素
LINDEX LINDEX key-name offset 返回列表元素中偏移量为offset的元素
LRANGE LRANGE key-name start end 返回列表元素中偏移量start到偏移量end的元素，其中包括start和end在内
LTRIM LTRIM key-name start end 对列表进行修剪，只保留start和end之间的数据，其中start和end的界值也会保留

阻塞式的列表弹出命令以及在列表之间移动元素的命令

命令 行为
BLPOP BLPOP key-name [key-name…] timeout 从第一个非空列表中弹出位于最左端的元素，或者在timeout秒之内阻塞并等待可弹出的元素出现
BRPOP BRPOP key-name [key-name…] timeout 从第一个非空列表中弹出位于最右端的元素，或者在timeout秒之内阻塞并等待可弹出的元素出现
RPOPLPUSH RPOPLPUSH source-key dest-key 从source-key列表中弹出最右端的元素，然后推入dest-key的最左端，并返回这个元素
BRPOPLPUSH BRPOPLPUSH source-key dest-key timeout 从source-key列表中弹出最右端的元素，然后推入dest-key的最左端，并返回这个元素，如果source-key位空，那么会在timeout秒之内阻塞并等待可弹出的元素出现

由上所述，我们可以看出使用这几个命令可以实现简单的消息队列。

问题点

我们知道Redis是单线程服务，而上述的命令BLPOP等是有阻塞时间的，当key-name中没有元素的时候会进行阻塞，如果阻塞客户端一直保持着跟服务器的链接，会不会阻塞其他命令的执行呢？

答案是不会的。这就涉及到redis阻塞命令的实现原理。我们知道，在redis server中有两个循环：IO循环和定时事件。在IO循环中，redis完成客户端连接应答、命令请求处理和命令处理结果回复等，在定时循环中，redis完成过期key的检测等。redis一次连接处理的过程包含几个重要的步骤：IO多路复用检测套接字状态，套接字事件分派和请求事件处理。

redis在blpop命令处理过程时，首先会去查找key对应的list，如果存在，则pop出数据响应给客户端。否则将对应的key push到blocking_keys数据结构当中，对应的value是被阻塞的client。当下次push命令发出时，服务器检查blocking_keys当中是否存在对应的key，如果存在，则将key添加到ready_keys链表当中，同时将value插入链表当中并响应客户端。

服务端在每次的事件循环当中处理完客户端请求之后，会遍历ready_keys链表，并从blocking_keys链表当中找到对应的client，进行响应，整个过程并不会阻塞事件循环的执行。所以， 总的来说，redis server是通过ready_keys和blocking_keys两个链表和事件循环来处理阻塞事件的。

Redis数据结构之字符串相关命令

这篇文章主要介绍的是Redis五大数据结构中的字符串常用的相关的命令

字符串之自增和自减

命令 行为
INCR INCR key-name 将键存储的值加上1
DECR DECR key-name 将键存储的值减1
INCRBY INCRBY key-name amount 将键存储的值加上整数amount
DECRBY DECRBY key-name amount 将键存储的值减去整数amount
INCRBYFLOAT INCRBYFLOAT key-name amount 将键存储的值加上浮点数amount

看到上图的几个命令可以看出即使value是字符串形式的数字，只要这个值可以被Redis解释成十进制整数或者浮点数的话就可以进行自增或者自减操作;如果输入的value无法被Redis解释成数字的话，进行自增或者自减就会出现错误。

字符串子串和二进制位

除了自增或者自减操作，Redis还可以对字符串的其中一部分内容进行读取或者写入操作

命令 行为
APPEND APPEND key-name value 将value追加到给定键key-name当前存储值的末尾
GETRANGE GETRANGE key-name start end 获取一个由偏移量start到偏移量end范围内所有字符组成的子串，包括start和end在内
SETRANGE SETRANGE key-name start value 将从start开始的子串设置为给定值
GETBIT GETBIT key-name offset 将字符串看成是二进制位串，并返回位串中偏移量为offset的二进制位的值
SETBIT SETBIT key-name offset value 将字符串看成是二进制位串，并将位串中偏移量为offset的二进制的值设置为value
BITCOUNT BITCOUNT key-name [start end] 统计二进制位串中面值为1的二进制位的数据，如果给定了start end范围，那么只计算该范围内的二进制位的数量
BITOP BITOP operation dest-key key-name [key-name …] 对一个或多个二进制位串进行包括并（AND）、或(OR)、异或(XOR)、非(NOT)在内的任意一种按位运算操作，并将计算得出的结果存放在dest-key里面

关于GETRANGE和SETRANGE两个命令需要说明的是

  • GETRANGE对字符串进行操作的时候，字符串从左到右是整数索引，从0开始，从右到左是负数作为，从-1开始，不支持回绕操作（就是 GETRANGE newstringkey -1 -15）,具体看下图：

  • SETRANGE如果这个key没有值，那么在设置start位置的时候，如果不是从0开始，那么前面的位置是需要补0的，如：SETRANGE newstringkey2 3 “this is new key and value”

我们知道 ‘a’ 的ASCII码是 97。转换为二进制是：01100001，setbit命令是对offset位置的值进行修改，比方说我们需要把’a’改成’b’，那么二进制是：01100010，我们需要将第六位设置为1，第七位设置为0，具体命令可以参考上图，最后使用get命令可以看到结果为’b’。

关于SETBIT和BITCOUNT的使用：可以用来计算用户上线的次数，使用用户名:id来作为key，用网站上线的日期到用户上线的日期作为offset，然后将这个值设置为1,SETBIT user:id offset 1,最后用GETBIT user:id来统计这个用户总共上线了多少次。

关于BITOP命令需要注意的是两个value是做的位运算！当BITOP处理不同长度的字符串时，较短的那个字符串所缺少的部分会被看做0,空的key也被看作是包含0的字符串序列

Redis数据结构

Redis分为五种数据结构

结构类型 结构存储的值 结构的读写能力
STRING 可以是字符串、整数、浮点数 对整个字符串或者字符串其中的一部分进行操作；对证书或者浮点数进行自增或者自减的操作
LIST 一个链表，每个链表的节点都包含了一个字符串 从链表的两端推入或者弹出元素；根据偏移量对链表进行修剪；读取单个或者多个元素；根据值查找或者移除元素
SET 包含字符串的无序收集器，并且包含的每个字符串都是独一无二各不相同的 添加、获取、移除单个元素；检查一个元素是否存在于集合中；计算交集、并集、差集；从集合中随机获取元素
HASH 包含键值对的无序散列表 添加、获取、移除单个键值对；获取整个键值对
ZSET 字符串成员与浮点数分值之间的有序映射，元素的排列顺序有分值大小决定 添加、获取、移除单个元素；根据分值范围或者成员来获取元素

集合简单操作

简单的几个命令

命令 行为
SADD 将给定元素添加到集合
SMEMBERS 返回给定集合中包含的所有元素
SISMEMBER 检查给定的元素是否存在于集合中
SREM 如果给定的元素存在于集合中，就删除这个元素

Redis集合和列表都可以存储多个字符串，他们的不同在于，列表可以存储多个相同的字符串，而集合通过散列表来保证自己存储的每个字符串都不相同

散列集合

简单的几个命令

命令 行为
HSET 在散列里面关联起给定的键值对

HGETALL 获取散列包含的所有键值对
HDEL 如果给定键存在于散列里面，那么移除这个键

有序集合

有序集合和散列一样都是用于存储键值对，有序集合的键被称为成员，每个成员都是各不相同的；有序集合的值被称为分值，分值必须为浮点数

简单的几个命令

命令 行为
ZADD 将一个带有分值的成员添加到有序集合中
ZRANGE 根据元素在有序排列中所处的位置，从有序集合里面获取多个元素
ZRANGEBYSCORE 获取有序集合在给定分值范围内的所有元素
ZREM 如果指定元素存在于有序集合中，那么移除这个元素

Linux中修改时区和时间的方法

查看和修改时区

  • 使用命令date -R查看时区

  • 使用命令tzselect修改时区，这时系统会让你选择洲、国家、城市

  • 选择完成之后复制相应的时区文件，替换系统时区文件

查看和修改时间

  • 使用date命令查看Linux上的系统时间

  • 使用命令 date -s 修改时间，可以只修改日期，也可以只修改时间，如果两者都要修改 格式为：date -s “2018-07-26 11:20:00”

  • 使用命令hwclock -w 将当前时间和日期写入BIOS，避免重启后失效

阅读GC日志

发表于 2018-03-16 | 更新于 2018-03-16 | 分类于 jvm

我们在工作中有时候会分析GC日志，虽然有可视化工具利器来查看，但我觉得普通GC日志还是要能看懂的，毕竟这也是属于我们技能的一部分。

JVM中打印出GC日志的参数包括以下几个：

-XX:+PrintGC 输出GC日志

-XX:+PrintGCDetails 输出GC的详细日志

-XX:+PrintGCTimeStamps 输出GC的时间戳（以基准时间的形式）

-XX:+PrintGCDateStamps 输出GC的时间戳（以日期的形式，如 2013-05-04T21:53:59.234+0800）

-XX:+PrintHeapAtGC 在进行GC的前后打印出堆的信息

-XX:+PrintGCApplicationStoppedTime // 输出GC造成应用暂停的时间

-Xloggc:../logs/gc.log 日志文件的输出路径
